# Security policy

## Supported version

Security fixes target the latest source release on `main`.

## Reporting

Do not publish API keys, private screenshots, source emails, or exploitable details in a public issue. Use GitHub's private vulnerability reporting for this repository. If that channel is unavailable, open a minimal issue asking the maintainer for a private contact channel without including the vulnerability details.

Include the affected version, reproducible conditions, impact, and a minimal synthetic test case. Reports involving renderer escape, unsafe URL fetching, secret exposure, unconsented network transmission, signature/notarization bypass, or fabricated verified provenance are treated as security issues.
